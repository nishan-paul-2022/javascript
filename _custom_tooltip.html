            Store Hotel facilities like Beach Side, Hill Side, Gym, Convention Hall, Swimming Pool
            Store Hotel information Like Name, Address, Image, Rating, Place Id (As a foreign key), [Store available hotel facilities] 
            Store Room Facilities Like Air Conditioner, Air Purifying, Smart TV, Furnished Bathroom, Parking Facility, High Chair
            Store Room Information like Image, Adult, Child, Bed, Hotel Id (As a foreign key), [Store available room facilities information as well] 
            
            - Tech stack : .net 6 + mvc 
            
            - Table List
            
            (a)
            table name : HotelFacility
            table fields : Id, Name
            table data : Beach side, Hill side, Gym, Convention hall, Swimming pool
            
            (b)
            table name : HotelInfo
            table fields : Id, Name, Address, Rating, Image, LocationId (as a foreign key from Location table primary key named Id) 
            
            (c)
            table name : RoomFacility
            table fields : Id, Name
            table data : Air conditioning, Air purifying, Smart TV, Furnished bathroom, Car parking, High chair
            
            (d)
            table name : RoomInfo
            table fields : Id, Adult, Children, Bed, Image, HotelId (as a foreign key from Hotel table primary key named Id)
            
            - Requirements
            
            01. Image field will contain the image file name.
            02. Image file name will be in this formation : imgId.ext
            03. Image field will be 16 character long.
            04. HotelFacility and HotelInfo have many to many relationship. That means HotelInfo must have information about facility that the hotel provides.
            05. RoomFacility and RoomInfo have many to many relationship. That means RoomInfo must have information about facility that the room provides.
            06. Rating filed value will be 1 to 5.
            07. Hotel Name will be 50 character length.
            08. Hotel Address will be 150 character length.
            09. Hotel Name and Address will support only alphanumeric character, punctuation mark and common special character.
            10. HotelFacility Name will be 50 character length.
            11. RoomFacility Name will be 50 character length.
            
            - What I want
            01. model for (a), (b), (c), (d) 
            02. every model will have validation on basis of requirements
            03. string length in model will be defined on basis of requirements
            04. seed 10 meaningful data to HotelInfo table on basis of  Location data 
            05. seed 20 meaningful data to RoomInfo table on basis of  HotelInfo data 
            
            (1) changes on location, corresponding changes on hotel and vice versa
            - when I delete a location, corresponding hotel data in hotel table will be removed
            - when I delete a hotel, hotel list of corresponding location data in location table will be updated
            
            (2) changes on hotel, corresponding changes on room and vice versa
            - when I delete a hotel, corresponding room data in room table will be removed
            - when I delete a room, room list of corresponding hotel data in hotel table will be updated
            
            (3) changes on hotel, corresponding changes on hotel-facility and vice versa
            - when I delete a hotel, corresponding hotel list of hotel-facility data in hotel-facility table will be updated
            - when I delete a hotel-facility, corresponding hotel-facility list of hotel data in hotel table will be updated
            
            (4) changes on hotel, corresponding changes on hotel-and-facility
            - when I delete a hotel, corresponding hotel-and-facility data in hotel-and-facility table will be removed 
            
            (5) changes on room, corresponding changes on room-facility and vice versa
            - when I delete a room, corresponding room list of room-facility data in room-facility table will be updated
            - when I delete a room-facility, corresponding room-facility list of room data in room table will be updated
            
            (6) changes on room, corresponding changes on room-and-facility
            - when I delete a room, corresponding room-and-facility data in room-and-facility table will be removed
